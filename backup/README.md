# Система бэкапа для сейсмической сети

Эта система предоставляет полный набор инструментов для создания резервных копий и восстановления базы данных сейсмической сети.

## Файлы

- `backup-system.js` - Скрипт для создания резервных копий
- `restore-system.js` - Скрипт для восстановления из резервных копий
- `auto-backup.js` - Автоматизированная система бэкапа
- `backups/` - Папка для хранения резервных копий

## Использование

### Создание резервной копии

```bash
# Создать резервную копию вручную
node backup/backup-system.js
```

### Восстановление из резервной копии

```bash
# Показать доступные резервные копии
node backup/restore-system.js

# Восстановить из конкретной резервной копии
node backup/restore-system.js seismic-backup-2025-01-17T12-00-00-000Z.sql
```

### Автоматизированные резервные копии

```bash
# Создать резервную копию немедленно
node backup/auto-backup.js backup

# Показать все резервные копии
node backup/auto-backup.js list

# Очистить старые резервные копии
node backup/auto-backup.js cleanup

# Запустить автоматический планировщик (ежедневно в 2:00)
node backup/auto-backup.js start
```

## Политика хранения

Автоматическая система бэкапа использует следующую политику хранения:

- **Ежедневные резервные копии**: 7 дней
- **Еженедельные резервные копии**: 4 недели
- **Ежемесячные резервные копии**: 12 месяцев

## Требования

- PostgreSQL клиент (`pg_dump` и `psql`)
- Переменная окружения `DATABASE_URL`
- Node.js для запуска скриптов

## Безопасность

- Резервные копии создаются в формате SQL
- Файлы содержат полную структуру и данные базы данных
- Храните резервные копии в безопасном месте
- Регулярно тестируйте процедуры восстановления

## Мониторинг

Система автоматически создает метаданные для каждой резервной копии:

- Временная метка создания
- Размер файла
- Тип резервной копии (ручная/автоматическая)
- Информация о базе данных

## Рекомендации

1. Запускайте автоматические резервные копии в производственной среде
2. Тестируйте процедуры восстановления регулярно
3. Храните резервные копии в нескольких местах
4. Мониторьте размер и успешность резервных копий
5. Документируйте все восстановления

## Устранение проблем

### Общие проблемы:

1. **Ошибка "DATABASE_URL not found"**
   - Убедитесь, что переменная окружения установлена

2. **Ошибка "pg_dump command not found"**
   - Установите PostgreSQL клиент

3. **Недостаточно места для резервной копии**
   - Очистите старые резервные копии
   - Увеличьте доступное место

4. **Ошибка восстановления**
   - Проверьте целостность файла резервной копии
   - Убедитесь, что база данных доступна

## Логи

Все операции записываются в консоль с временными метками и статусами:

- ✅ Успешные операции
- ❌ Ошибки
- 🔄 Выполняющиеся процессы
- 📊 Информация о размерах
- 🧹 Операции очистки
