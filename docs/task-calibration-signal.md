# Калибровка датчиков и обработка сигналов

## What & Why
Реализовать модуль калибровки сейсмометров (включая АЧХ) и модуль регистрации/обработки сигналов: пороговое срабатывание по компонентам, запись волновых форм и спектральный анализ (FFT, H/V-метод Накамуры). Это ключевые инструменты полевого сейсмолога.

## Done looks like
- Страница "Калибровка" позволяет создать сессию калибровки для выбранного датчика: ввести коэффициент чувствительности по осям Z/N/E, загрузить или нарисовать АЧХ (таблица частота → амплитуда), сохранить в БД
- График АЧХ отображается как интерактивная кривая на Chart.js/D3 с логарифмической осью частот
- Список прошлых калибровок с датами; предупреждение если калибровка просрочена (>6 мес)
- Страница "Записи событий" показывает трёхкомпонентные волновые формы (Z, N, E) для каждого зарегистрированного события
- Пороги срабатывания настраиваются отдельно для вертикальной и горизонтальной компонент (в мм/с или долях g)
- Страница "Спектральный анализ": кнопка "Вычислить FFT" по выбранной записи → отображение амплитудного спектра
- H/V метод: расчёт отношения горизонтальной к вертикальной компоненте спектра, отображение кривой H/V с пиком резонансной частоты площадки

## Out of scope
- Реальный SeedLink-протокол (используется симуляция / загрузка файла)
- Расчёт МТСМ и откликов конструкций (отдельная задача)

## Steps
1. **Таблицы калибровки в схеме** — Добавить `calibration_sessions` (датчик, дата, оператор, коэффициенты по осям) и `calibration_afc` (точки АЧХ: частота, амплитуда, фаза) в `shared/schema.ts`.
2. **API калибровки** — Endpoints для создания сессии, сохранения АЧХ, получения истории калибровок по датчику.
3. **UI калибровки** — Форма новой сессии + интерактивный редактор таблицы АЧХ + график кривой на D3/Chart.js с логарифмической шкалой частот.
4. **Таблица записей событий** — Добавить поля `component_z`, `component_n`, `component_e` в `waveform_data` или отдельную таблицу `signal_records`; API и страница с трёхкомпонентным графиком.
5. **Настройка порогов** — UI для задания пороговых значений по компонентам для каждого объекта/датчика; хранение в БД.
6. **FFT-анализ** — Реализация FFT на клиенте (библиотека fft.js или встроенная Web Audio API), отображение амплитудного спектра выбранной записи.
7. **H/V метод** — Расчёт спектральных отношений H/V по трём компонентам, визуализация кривой с выделением пика резонансной частоты.
