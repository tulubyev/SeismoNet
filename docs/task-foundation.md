# Фундамент: БД и новая структура проекта

## What & Why
Переориентировать базу данных и ядро приложения с регионального мониторинга природных землетрясений на инженерно-сейсмологическую систему мониторинга городской инфраструктуры Иркутска. Новая схема должна поддерживать объекты инфраструктуры, грунтовые профили и привязку датчиков к конкретным зданиям.

## Done looks like
- В БД появились таблицы: `infrastructure_objects` (здания/сооружения с параметрами конструкции), `soil_profiles` (типы и характеристики грунтов), `soil_layers` (слои разреза с Vs, Vp, плотностью), `sensor_installations` (привязка датчика к объекту)
- Страница "Объекты мониторинга" показывает список зданий/сооружений с привязанными датчиками
- Страница "Dashboard" переработана — фокус на Иркутск: карта города на Leaflet с объектами мониторинга
- Старые глобальные разделы (USGS/JMA/EMSC) переименованы или скрыты, добавлен фильтр "Иркутск / региональные данные"
- Навигация сайдбара переструктурирована под новые разделы: Объекты, Грунты, Сигналы, Расчёты, ПН

## Out of scope
- Расчёты усиления и отклика конструкций (отдельная задача)
- Интерфейс калибровки (отдельная задача)
- FFT и обработка сигналов (отдельная задача)

## Steps
1. **Новые таблицы схемы** — Добавить `infrastructure_objects`, `soil_profiles`, `soil_layers`, `sensor_installations` в `shared/schema.ts` с нужными полями (тип конструкции, этажность, год, материал; классификация грунта по СП 14.13330, Vs30, глубина слоёв).
2. **Обновить storage.ts и routes** — CRUD API для новых таблиц, привязка датчика к объекту через `sensor_installations`.
3. **Переработать дашборд** — Карта Иркутска (центр: 52.29°N, 104.30°E, zoom 12) с маркерами объектов мониторинга вместо глобальной карты землетрясений. Виджеты: активные объекты, последние события, статус сети.
4. **Страница объектов инфраструктуры** — Таблица объектов с поиском, фильтрами по типу (жилое/промышленное/мост/трубопровод), ссылками на привязанные датчики.
5. **Обновить навигацию сайдбара** — Разделы: Обзор (ПН), Объекты мониторинга, Датчики, Грунты, Сигналы, Расчёты, Настройки.
6. **Миграция БД** — Запустить `npm run db:push` для применения новой схемы к VPS-базе `seismonet_db`.
