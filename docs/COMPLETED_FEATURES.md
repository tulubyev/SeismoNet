# Выполненные дополнения

Документ фиксирует все реализованные функции в хронологическом порядке.
Обновляется по мере завершения задач.

---

## Этап 1 — Фундамент (апрель 2025)

### #1 · Фундамент: новая БД и структура для Иркутска
Переориентация базы данных с мониторинга природных землетрясений на инженерно-сейсмологическую систему мониторинга городской инфраструктуры Иркутска.
- Новая схема PostgreSQL: объекты инфраструктуры, грунтовые профили, сенсорные установки, расчёты
- Роли пользователей: администратор, пользователь, наблюдатель
- Портальная главная страница с блоками навигации
- Временный обход авторизации для разработки (`dev_superadmin`)

### #2 · Калибровка датчиков, АЧХ и обработка сигналов
Модуль калибровки сейсмометров и регистрации/обработки сигналов.
- Страница калибровки: сессии, АЧХ-кривые, параметры датчика
- Пороговое срабатывание по компонентам Z/NS/EW
- Запись волновых форм и FFT-спектральный анализ
- H/V-метод Накамуры для оценки частоты резонанса грунта

### #3 · База данных грунтов и объектов Иркутска
Раздел «Грунты» с геологическими условиями в точках наблюдения.
- Типы грунтов по СП 14.13330.2018, скоростные профили (Vs, Vp)
- Данные буровых скважин и слоёв
- Интерактивная карта объектов инфраструктуры на Leaflet
- Привязка грунтовых профилей к объектам наблюдения

### #4 · Расчёт усиления (МТСМ) и отклика конструкций
Инженерно-расчётный модуль сейсмического анализа.
- Метод тонкослоистых сред (МТСМ): 1D расчёт усиления сигнала в грунтовом разрезе
- Расчёт спектров ответа конструкций
- Прогноз резонансных условий «грунт–здание»
- Каталог байкальских землетрясений как входные данные

## Этап 2 — Спектральный анализ и расчёты (апрель 2026)

### #15 · Библиотека акселерограмм СП 14.13330
Синтетические нормативные акселерограммы по СП 14.13330.2018.
- 12 записей для разных категорий сейсмичности и грунтов
- Синтез методом случайной фазы с огибающей и полосовым фильтром
- Выбор записи прямо в интерфейсе вкладки «Отклик»

### #16 · Журнал расчётов
Браузер истории всех сохранённых расчётов конструкций.
- Список с типом, датой, объектом, грунтовым профилем
- Просмотр подробностей: входные параметры, результирующая кривая
- Экспорт в CSV
- Фильтрация по типу расчёта и поиск

### #17 · Нормативный спектр СП 14 на графике отклика
Наложение нормативной формы спектра СП 14 на расчётный график.
- Голубая пунктирная кривая «Sa норм. СП 14» поверх расчётного спектра
- Автоматический расчёт по выбранным K₁, категории грунта, PGA
- Аннотация с формулой и параметрами под графиком

### #18 · Расчёт по трём компонентам с усреднением
Спектр отклика рассчитывается одновременно по Z, NS, EW.
- Три отдельные кривые компонент на графике
- Геометрическое среднее как результирующий «расчётный спектр»
- Режим выбора: одна компонента / три / геомеана горизонталей

### #19 · Сравнение двух сохранённых расчётов
Наложение 2–3 расчётных кривых на один график в диалоге сравнения.
- Цветовое кодирование кривых (синий / красный / фиолетовый)
- Выбор расчётов чекбоксами в журнале (только одного типа)
- Экспорт объединённого CSV

### #20 · Инженерные заметки к расчётам
Текстовые заметки, прикреплённые к каждому сохранённому расчёту.
- Редактор с кнопкой сохранения
- Заметки видны в деталях расчёта и в карточке сравнения

### #21 · Регулируемый коэффициент K₁ (допускаемые повреждения)
Ползунок K₁ на вкладке «Отклик» по Таблице 5 СП 14.
- 5 категорий: нормальный 1.0, повышенный 0.8, особый 0.5 и др.
- Нормативный спектр и расчётный Sa пересчитываются в реальном времени

### #22 · Индивидуальные кривые Z, NS, EW рядом с усреднённой
Три компонентных спектра отображаются тонкими линиями наряду с геосредним.
- Z — серая, NS — зелёная, EW — оранжевая линия
- Легенда подписывает каждую компоненту и итоговый геосредний спектр

## Этап 3 — Горизонтальные пары H1/H2 и сравнения (апрель 2026)

### #23 · Горизонтальная пара H1/H2 для записей СП 14
Синтез двух статистически независимых горизонтальных компонент из одной нормативной записи.
- H1 и H2 с разными фазами, одинаковой целевой амплитудой
- Геометрическое среднее H1⊕H2 как расчётный спектр (режим GMH)
- Переключатель «Одиночная / H1⊕H2» в интерфейсе

### #25 · Метрики разницы между кривыми сравнения
Количественные дельты между сравниваемыми расчётами.
- Δ частоты пика (МТСМ) или Δ периода пика (спектр отклика)
- Отношение амплитуд пиков и интегральных площадей
- Полоска статистики над графиком и строки summary в CSV

### #26 · Сохранение и публикация наборов сравнения
Именованные наборы сравнения, доступные всем пользователям.
- Таблица `comparison_sets` в БД (название, тип, ID расчётов)
- Панель «Сохранённые сравнения» на странице расчётов
- Кнопки: «Открыть», «Скопировать ссылку», «Удалить»
- Глубокая ссылка `/calculations?compare=12,17` — автооткрытие диалога

### #27 · Аудит правок инженерных заметок
Фиксация автора и времени последней правки заметки.
- Колонки `notes_updated_at` и `notes_updated_by` в таблице расчётов
- Строка «изменено пользователем X · дата» под полем заметки

### #28 · Коэффициент K₂ (тип конструктивной системы)
Второй нормативный коэффициент по Таблице 6 СП 14.
- 5 категорий: рамная без связей 1.5, рамная со связями 1.3, стеновая монолитная 1.0, деревянная 0.8, каменная защищённая 0.7
- Выбирается рядом с K₁; нормативный спектр пересчитывается мгновенно
- Сохраняется вместе с расчётом

### #29 · Индикатор доминирующей компоненты на пике спектра
Цветной бейдж и умный тултип показывают, какая компонента вносит максимальный вклад в пике.
- Бейдж «▲ NS @ T=0.42с» в заголовке карточки графика
- Тултип разделён на раздел компонент (отсортирован по убыванию) и раздел агрегатов

### #30 · Горизонтальная пара H1/H2 для каталога Байкала
Та же логика GMH-пары, применённая к синтетическим акселерограммам каталога байкальских сценариев.
- H1/H2 с разными фазовыми тегами для каждого сценария
- Переключатель «Одиночная / H1⊕H2» в блоке каталога
- Компонента сохраняется в параметрах расчёта для аудита

### #31 · Индикатор разброса H1/H2
Панель с тремя метриками рассеивания между горизонтальными компонентами.
- Пиковое, медианное и среднее отношение Sa_H1/Sa_H2 по всем периодам
- Цветовая индикация: зелёный (<1.3×), янтарный (1.3–1.6×), красный (>1.6×)
- Интерпретирующее сообщение: малый / умеренный / высокий разброс
- Столбец `H1_H2_ratio` в экспортируемом CSV

### #32 · Визуальные аннотации пиков на графике сравнения
Вертикальные линии и закрашенные области между пиками кривых.
- Пунктирная цветная линия на частоте/периоде пика каждого расчёта
- Полупрозрачная заливка зоны между базовым и сравниваемым пиком
- Чекбокс «Аннотации пиков» для скрытия

### #33 · Полная история правок заметок
Хронологический журнал всех версий инженерских заметок к расчёту.
- Таблица `calculation_note_history` в БД
- Панель «История» в редакторе заметок (открывается по кнопке)
- Кнопка «Восстановить» для возврата к любой предыдущей версии
- API: `GET /api/calculations/:id/note-history`, `POST .../revert`

## Этап 4 — Улучшения графиков и сохранение параметров (апрель 2026)

### #34 · Сохранение K₁/K₂ для каждого здания
Коэффициенты K₁ и K₂ теперь хранятся у объекта инфраструктуры и загружаются автоматически.
- Колонки `k1Key` и `k2Key` добавлены в таблицу `infrastructure_objects`
- Выпадающий список зданий на вкладке «Отклик» — при выборе здания K₁/K₂ подставляются автоматически
- При изменении K₁/K₂ значения тут же сохраняются обратно в карточку здания
- Сохранённый расчёт включает ID здания и имя для полной прослеживаемости

### #35 · Таблица Sa по ключевым инженерным периодам
Компактная таблица значений Sa для стандартных периодов T = 0.1, 0.2, 0.5, 1.0, 2.0 с.
- Отдельный столбец для каждой компоненты (Z, NS, EW, H1, H2) + итоговый «Sa расч.»
- Строка ближайшая к пику выделена фиолетовым фоном с меткой «▲ пик»
- Доминирующая компонента в строке пика отмечена звёздочкой ★
- Кнопка «Копировать» — таблица копируется как текст для вставки в Excel/Word
