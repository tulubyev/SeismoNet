# Спецификация ролей пользователей системы

## Сеть сейсмических наблюдений за объектами инфраструктуры г. Иркутска

---

## Роли системы

### 1. Суперадмин (`superadmin`)
**Все функции системы без ограничений.**
- Управление пользователями, назначение ролей
- Настройка системы, интеграций, оповещений
- Доступ ко всем модулям: мониторинг, расчёты, грунты, нормативы, устройства
- Просмотр журналов, экспорт всех данных
- Управление объектами инфраструктуры и датчиками

---

### 2. Конструктор (`designer`)
**Разрабатывает схему расположения датчиков в соответствии со структурной схемой здания.**
- Работа с 3D-схемой здания / структурной схемой
- Размещение и конфигурирование датчиков на объекте (позиции, этажи, типы)
- Просмотр нормативной базы (СП, СНиП, ГОСТ) — нормы сейсмостойкости
- Расчёт схем мониторинга здания
- Только чтение: данные мониторинга, сейсмограммы
- Нет доступа: настройки устройств, администрирование

---

### 3. Сейсмолог (`seismologist`)
**Анализирует сейсмические процессы и данные наблюдений.**
- Полный доступ к сейсмограммам: просмотр, разметка P/S волн, экспорт
- Спектральный анализ (FFT, H/V Накамуры, АЧХ)
- Просмотр и редактирование каталога событий
- Карта сейсмичности: просмотр и аннотации
- Работа с профилями грунтов (чтение + внесение данных бурения)
- Расчёт усиления сигнала (МТСМ)
- Только чтение: объекты инфраструктуры, устройства
- Нет доступа: управление пользователями, настройки системы

---

### 4. Data Analyst (`data_analyst`)
**Анализирует расчётные данные: спектры, параметры колебаний, отклики конструкций.**
- Доступ к результатам расчётов (МТСМ, спектры ответа, резонансные частоты)
- Просмотр и выгрузка данных сейсмограмм
- Построение графиков, статистика событий
- Работа с базой данных грунтов (только чтение)
- Экспорт данных в CSV / JSON
- Только чтение: всё остальное
- Нет доступа: настройка устройств, управление системой

---

### 5. Device Manager (`device_manager`)
**Отвечает за работу датчиков и их настройку.**
- Управление датчиками: добавление, редактирование параметров, статус
- Калибровка датчиков, настройка АЧХ и порогов срабатывания
- Диагностика: батарея, качество сигнала, связь, хранилище
- Настройка триггеров и режимов записи
- Только чтение: сейсмограммы, события, объекты
- Нет доступа: расчётные модули, нормативная база, управление пользователями

---

### 6. Staff (`staff`)
**Представитель застройщика или муниципалитета.**
- Просмотр данных по своим объектам инфраструктуры
- Просмотр отчётов и состояния мониторинга объекта
- Просмотр карты и статуса сети (только чтение)
- Нет доступа: сейсмограммы, расчёты, конфигурация датчиков, нормативная база в полном объёме, администрирование
