# Пункт наблюдения — онлайн-концентратор данных

## What & Why
Реализовать главный интерфейс пункта сейсмического наблюдения города Иркутска: онлайн-концентратор всех данных в реальном времени, мониторинг объектов, система оповещений, экспорт данных в стандартных форматах. Это оперативный рабочий стол дежурного сейсмолога.

## Done looks like
- Страница "Пункт наблюдения" (главная, доступна без авторизации в режиме просмотра) со сводной панелью: карта Иркутска с цветными маркерами объектов (зелёный=норма, жёлтый=превышение порога, красный=тревога)
- Лента событий в реальном времени: последние срабатывания порогов по объектам, сортировка по времени
- Виджет "Текущий уровень сейсмичности": максимальная амплитуда за последние 24 часа по всем датчикам
- Режим "Живой сигнал": при клике на объект открывается панель с реальным (симулированным) графиком трёх компонент в реальном времени через WebSocket
- Система оповещений: при превышении порога — уведомление в Telegram и по email (через уже настроенные интеграции)
- Страница "Экспорт данных": выбор объекта, периода, формата (CSV с временными рядами; MiniSEED-совместимый JSON), кнопка скачать
- Страница "Журнал событий": хронологический список всех превышений порогов с деталями (объект, компонента, амплитуда, длительность)
- Мобильная версия для дежурного: компактный дашборд с лентой тревог и быстрым просмотром сигнала

## Out of scope
- Реальный прием данных по SeedLink (используется WebSocket-симуляция)
- Интеграция с МЧС или государственными системами оповещения

## Steps
1. **Страница Пункта наблюдения** — Новый маршрут `/monitoring-center`, карта Иркутска с объектами, цветовая индикация статуса, лента событий слева.
2. **Реальный WebSocket для живого сигнала** — Обновить серверный WebSocket: добавить тип сообщения `LIVE_SIGNAL` с симулированными трёхкомпонентными данными для выбранного объекта; клиент отображает прокручивающийся график.
3. **Страница журнала событий** — Хронологическая таблица срабатываний с фильтрами по объекту, дате, компоненте; детальная карточка события с мини-графиком волновой формы.
4. **Экспорт данных** — Форма выбора объекта + период + формат; серверный endpoint для генерации и отдачи файла CSV или JSON.
5. **Оповещения при превышении порога** — Интегрировать проверку порогов в серверный цикл; при срабатывании — отправить Telegram-уведомление (через уже существующий `server/services/telegram.ts`) с указанием объекта, компоненты, значения.
6. **Мобильный дашборд ПН** — Переработать `MobileDashboard.tsx`: лента последних тревог, статус объектов, кнопка "Срочный отчёт".
7. **Статус-страница (публичная)** — Простая страница `/status` без авторизации: общий статус сети (N объектов онлайн, последнее событие).
